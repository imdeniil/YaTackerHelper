# Принципы организации кода

## Модульность
- Разбивай большие файлы (>300 строк) на логические модули
- Один файл = одна ответственность (Single Responsibility Principle)
- Группируй связанный код в пакеты (директории с `__init__.py`)
- Используй `__init__.py` для реэкспорта публичного API пакета

## Структура пакетов
```
module/
├── __init__.py      # Реэкспорты публичного API
├── core.py          # Основная логика модуля
├── helpers.py       # Вспомогательные функции
├── decorators.py    # Декораторы (если есть)
├── exceptions.py    # Кастомные исключения (если нужны)
└── constants.py     # Константы и конфигурация (если нужны)
```

## Разделение по слоям
- **routes/handlers** - точки входа (HTTP endpoints, bot handlers)
- **services** - бизнес-логика
- **database/crud** - работа с данными
- **components/views** - UI компоненты (если есть фронтенд)
- **utils** - общие утилиты

## Статические файлы
- Выноси JS/CSS в отдельные файлы (`static/js/`, `static/css/`)
- Не встраивай большие скрипты/стили в код

## Именование
- `setup_*()` или `init_*()` - функции инициализации/регистрации
- `_private_function()` - приватные функции модуля (с underscore)
- `UPPER_CASE` - константы
- `snake_case` - функции и переменные
- `PascalCase` - классы
