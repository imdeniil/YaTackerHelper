# Начало работы 
1. Проверь есть ли виртуальное окружение .venv, если его нет создай
2. Проверь установлены ли зависимости uv sync 
3. Для публикации на pypi и сборки пакетов используй uv 

# Принципы документирования:
0. Все файлы документации храняться в папке docs
1. НЕ ПОВТОРЯЙСЯ В ДОКУМЕНТАЦИИ, НЕ ДУБЛИРУЙ ЕЁ, ССЫЛАЙСЯ НА ТО ГДЕ ЧТО МОЖНО НАЙТИ
2. Перечень файлов документации не является конечным, при необходимости может быть расширен и дополнен 
3. Если не найдены ключевые файлы перечисленные в "Файлы документации", создай их на основе кода с которыми работаешь
4. В каждом файле документации верстай подробное оглавление и используй поиск для ориентации в них, не читай файлы полностью, если в этом нет необходимости. 
5. При добавлении новых классов или файлов обновляй CLASSES_and_FILES.md
6. При внесении значительных изменений в кодовую базу обновляй CHANGELOG.md и TRANSIT.md
7. При внедрении Архитектурных принципов, Технологического стека, Правил стилизации кода обновляй TECH.md 

# Файлы документации
CHANGELOG.md - Глобальные версионные изменения, формат основан на https://keepachangelog.com/en/1.0.0 и https://semver.org/spec/v2.0.0.html

TRANSIT.md - Промежуточный контекст который обновляется по мере необходимости при разработке, в нем должна содержаться информация которая позволит быстро сориентироваться в актуальном состоянии проекта и продолжить работу, при начале нового чата. Будь в нем краток, удаляй то что уже однозначно доделано, оставляй только то над чем необходимо продолжать работу в ближайшие шаги. В начале содержит концептуальное описание проекта.

CLASSES_and_FILES.md - Содержит структуру классов проекта

TECH.MD - Содержит Архитектурные принципы, Технологический стек, Правила стилизации кода

README.md - Обновляется только по прямому запросу
